# 🫀 VitalGraph

**Health intelligence through graph-connected ML — visualized as a living plant.**

VitalGraph transforms raw patient health data into personalized, actionable health suggestions using XGBoost, K-Means clustering, and anomaly detection. Each suggestion maps to a leaf on an interactive plant visualization — complete the suggestion, and your flower blooms.

Built at **Hack w/ Chicago 2026**.

---

## ✨ How It Works

```
Patient Data ──► XGBoost Models ──► Per-Feature Risk Contributions ──► Category Health Scores
                                                                              │
                                                                              ▼
                                                           RocketRide / Claude LLM
                                                                              │
                                                                              ▼
                                                         Personalized Suggestions
                                                                              │
                                                                              ▼
                                                    Interactive Plant Leaves 🌿🌸
```

1. **Sample a patient** from a 100K-row diabetes dataset
2. **Run 4 trained ML models** — XGBoost classifier, risk regressor, K-Means, Isolation Forest
3. **Extract per-patient feature contributions** (SHAP-like `pred_contribs`) from XGBoost
4. **Group features into 5 health categories** (Nutrition, Exercise, Sleep, Stress, Monitoring) and compute risk shares
5. **Generate personalized recommendations** via RocketRide pipeline or Claude API
6. **Render as interactive leaves** on a plant — leaf color reflects health score, completing tasks blooms the flower

## 🧠 ML Pipeline

| Model | Type | Purpose |
|-------|------|---------|
| **XGBoost Classifier** | Binary classification | Predicts diabetes diagnosis probability |
| **XGBoost Regressor** | Regression | Predicts diabetes risk score (0-100) |
| **K-Means (k=5)** | Unsupervised clustering | Assigns patient to archetype cluster |
| **Isolation Forest** | Anomaly detection | Flags anomalous patient profiles |
| **XGBoost Stress Classifier** | Binary classification | Classifies wearable data as stressed/non-stressed |

### Scoring — No If-Else

Category health scores are computed entirely from the XGBoost regressor's **`pred_contribs`** — tree-based per-feature contributions for each individual patient:

```
risk_share(category) = Σ |contribution(feature)| for features in category
                       ─────────────────────────────────────────────────
                       Σ |contribution(feature)| for ALL features

health_score = sigmoid(risk_share) → 0-100
```

### Datasets

| Dataset | Records | Source |
|---------|---------|--------|
| `diabetes_dataset.csv` | 100,000 patients | Vitals, labs, lifestyle, diabetes staging |
| `activity_summary.xlsx` | 138 rows | Apple Watch, Garmin, Fitbit signal quality benchmarks |
| `Wearable_Dataset/` | Variable | PhysioNet E4 wristband data (STRESS / AEROBIC / ANAEROBIC) |

## 🌿 Plant Visualization

The frontend renders a living plant where **each leaf = one health suggestion**:

- **Leaf color** reflects the ML health score (🟢 green = healthy → 🟡 amber = needs attention)
- **Leaf count** is dynamic — more suggestions = more leaves
- **Tapping a leaf** shows the suggestion detail with:
  - Circular score ring
  - Patient metric cards
  - 🤖 ML Model Insight badge (risk contribution %, top driving feature)
  - AI-generated recommendation text
- **Completing a task** marks the leaf with a ✓ and **blooms the flower** incrementally
- **Sun**, **clouds**, **roots**, and **pot** are also interactive with health-state-driven animations

## 🛠 Tech Stack

### Backend
- **Python 3** — FastAPI server
- **XGBoost** — trained classifier + regressor with `pred_contribs`
- **scikit-learn** — K-Means clustering, Isolation Forest, StandardScaler
- **RocketRide** — AI pipeline orchestration (with Anthropic Claude fallback)
- **Neo4j** — graph database for patient → risk profile → cluster relationships
- **Pandas / NumPy** — data processing

### Frontend
- **React 18** + **TypeScript** — Vite-based SPA
- **Framer Motion** — plant animations, leaf sway, flower bloom
- **Tailwind CSS 4** — styling
- **Lucide React** — icons

### Data Science (Notebook)
- **Jupyter** — `vitalgraph_main.ipynb` for full training pipeline
- **Plotly / Matplotlib / Seaborn** — EDA visualizations
- **Streamlit** — `app.py` dashboard (alternative UI)

## 🚀 Getting Started

### Prerequisites

- Python 3.9+
- Node.js 18+
- Trained models file (`models.pkl` — generated by running the notebook)

### 1. Clone & Install

```bash
git clone https://github.com/yourusername/VitalGraph.git
cd VitalGraph

# Python dependencies
pip install fastapi uvicorn xgboost scikit-learn pandas numpy anthropic python-dotenv rocketride

# Frontend dependencies
cd frontend && npm install && cd ..
```

### 2. Environment Variables

Create a `.env` file in the project root:

```env
# Required for LLM recommendations
ANTHROPIC_API_KEY=sk-ant-...

# Optional — RocketRide pipeline
ROCKETRIDE_URI=http://localhost:5565
ROCKETRIDE_APIKEY=your-key

# Optional — Neo4j graph database
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your-password
```

### 3. Train Models (if `models.pkl` doesn't exist)

Open and run all cells in `vitalgraph_main.ipynb`. This trains XGBoost, K-Means, and Isolation Forest models, then saves them to `models.pkl`.

### 4. Run

```bash
# Terminal 1 — Backend API
uvicorn api:app --reload --port 8000

# Terminal 2 — Frontend
cd frontend && npm run dev
```

Open **http://localhost:5173** to see the plant.

### 5. Streamlit Dashboard (alternative)

```bash
streamlit run app.py
```
